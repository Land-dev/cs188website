SLAM-Based Firefighting Drone Simulation

Ryan Teoh, Ayaan Raina, Yash Goyal
University of California, Los Angeles
Github

Abstract

Autonomous drones can assist firefighting operations by exploring hazardous environments and locating fire sources without risking human responders. This project develops a simulated autonomous firefighting drone capable of mapping unknown environments, localizing itself, and detecting fires. The system is implemented in the PyBullet physics simulator using the gym-pybullet-drones framework. A LiDAR sensor is simulated using radial ray casting to generate range measurements, which are used to construct a 2D occupancy grid map of the environment. Drone localization is performed using dead reckoning combined with an Extended Kalman Filter (EKF) to fuse motion predictions with noisy sensor observations. To detect fires, a computer vision model processes images from a simulated onboard camera to identify fire regions in the environment. Once a fire is detected, its location is recorded in the map and the drone navigates toward the target using grid-based path planning. Sensor noise is incorporated into position and LiDAR measurements to simulate realistic operating conditions. Experimental results demonstrate that the system can autonomously explore an unknown environment, build a map, detect fires using vision, and navigate safely toward detected fire locations.

Methods

Simulation Environment

The system is implemented in the PyBullet physics engine using the gym-pybullet-drones framework. A quadrotor drone model is simulated with realistic dynamics and controlled using a PID-based flight controller. The controller stabilizes the drone and enables waypoint tracking during exploration and navigation. Indoor environments with walls and obstacles are created to represent cluttered spaces where fires may occur.

LiDAR Sensing and Mapping

A planar LiDAR sensor is simulated using radial ray casting from the drone's position. Each ray returns the distance to the nearest obstacle, which is projected into the global frame using the drone's estimated pose. These measurements are used to update a 2D occupancy grid map, allowing the drone to build a representation of the environment while exploring.

Localization

The drone's pose is initially predicted using dead reckoning, where motion is estimated from control inputs and a simulated IMU. An Extended Kalman Filter (EKF) is used to fuse these predictions with sensor observations and reduce drift. Sensor noise is introduced to improve realism, including Gaussian noise in position measurements and LiDAR range readings. The EKF incorporates process noise to account for uncertainty in the drone's motion model and produces a filtered estimate of the drone's state.

Fire Detection Using Computer Vision

Instead of detecting fires using ray casting, the system uses a computer vision model operating on images captured from the drone's onboard camera. The model analyzes RGB images to identify fire regions within the scene. When a fire is detected, its location is estimated relative to the drone and projected into the global map frame. This allows detected fires to be stored within the occupancy map and treated as navigation targets.

Exploration and Navigation

During the exploration phase, the drone moves through the environment while updating the occupancy map. Once a fire is detected, the system switches to goal-directed navigation. A grid-based A* path planner computes a collision-free path through the occupancy grid from the drone's current position to the detected fire location. The PID controller then tracks the planned waypoints until the drone reaches and hovers over the target location.
